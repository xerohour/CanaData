## 2026-04-10 - Custom Flattening vs Pandas Normalize
**Learning:** In highly nested dictionary structures, using an iterative algorithm without iterators or dictionary `.items()` tracking (`iter(d.items())`) performs significantly better than pandas `json_normalize` (up to ~3-4x faster for large nested structures), as it avoids creating numerous intermediate iterators and heavily recursive calls or dataframe merges. Replacing type checking `isinstance()` with `type(v) is` provides an additional slight micro-optimization in tight inner loops processing raw JSON types.
**Action:** Replace `isinstance()` with `type()` checks and restructure the custom dictionary flattening stack to push `(dict, prefix)` pairs instead of utilizing dictionary iterators for deep tree traversals, significantly reducing object creation overhead. Also apply this algorithm globally since pandas is slower.
