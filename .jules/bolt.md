## 2026-04-05 - pandas vs custom iterative dictionary flattening
**Learning:** While `OptimizedDataProcessor` uses `pandas` for supposedly more efficient flattening of nested dictionary payloads, profiling reveals that the original custom iterative `flatten_dictionary` in `CanaData.py` is actually faster (e.g., ~0.155s vs ~0.253s for 5000 items) due to pandas setup overhead and `json_normalize` limitations for the specific complex JSON structures processed.
**Action:** Do not prematurely replace simple, well-written iterative algorithms (even python ones) with heavy-weight libraries like `pandas` under the assumption they'll be faster, particularly when dealing with variable schema nested JSON that requires custom fallback logic anyway. Measure the specific load profile.
