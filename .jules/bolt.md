## 2026-03-25 - Pre-computing row strings in CanaParse
**Learning:** In CanaParse, redundant string joining within the inner loop of apply_filters (filters x rows) is optimized by pre-calculating a lower-cased row_str for each row before the filter iteration begins, resulting in measurable performance gains (~15-57%). Also avoid copying every row to evaluate its condition, as creating a shallow copy of every row prior to condition evaluation causes severe memory and performance regressions. Evaluate the condition on the original row and only append a copy of it upon a successful match, preserving the original list comprehension's evaluation order.
**Action:** When working on array filtering loops, precompute shared string representations or other expensive operations globally rather than in the inner match conditions. Evaluate conditions on references rather than shallow copies.
