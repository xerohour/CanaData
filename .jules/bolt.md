## 2025-02-28 - Optimizing Pandas JSON normalization and dictionary conversions
**Learning:** Pandas `to_dict('records')` introduces huge overhead on wide datasets due to dataframe/series conversion checks. Similarly, `df[col].apply` suffers from Series overhead for simple element-wise operations like stringification. However, avoiding `to_dict('records')` by using `itertuples()` is dangerous because `itertuples` preserves underlying NumPy scalar types (like `np.int64`), which will cause JSON serialization failures downstream.
**Action:** Replace `apply(lambda x: ...)` with `[func(x) for x in df[col]]` to bypass Pandas Series overhead for element-wise string formatting. When converting DataFrames to dictionaries for JSON serialization pipelines, stick with `to_dict('records')` despite its overhead, as it safely casts NumPy types back to native Python types. Avoid partial inline JSON pre-serialization logic that only covers top-level keys if `pd.json_normalize` handles nested objects anyway.
