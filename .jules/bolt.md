## 2024-05-20 - Prevent Proactive Copying in List Comprehensions
**Learning:** In `CanaParse.apply_filters`, proactively copying every row (`row[:]`) before passing it to `is_match` to prevent mutating `self.raw_data` causes severe memory and performance regressions, as Python allocates a new list for every single row in the dataset regardless of whether it matches the filter condition or not.
**Action:** Evaluate the condition on the original row and only append a copy of it upon a successful match (`filtered.append(original_row[:])`). This can be done by replacing the list comprehension `[row[:] for row in self.raw_data if self.is_match(row, ...)]` with a standard `for` loop that checks `is_match(row, ...)` first, then appends `row[:]`. Oh wait, `[row[:] for row in self.raw_data if self.is_match(row, ...)]` actually evaluates `is_match(row, ...)` FIRST before evaluating `row[:]`. Why does the journal entry say "Do NOT proactively copy every row (row[:]) before passing it to is_match"? The list comprehension `[row[:] for row in self.raw_data if self.is_match(row, f, price_col)]` DOES NOT copy the row before passing it to `is_match`. It evaluates `is_match(row)` first, and if True, it evaluates `row[:]`.

> "In CanaParse.apply_filters, do NOT proactively copy every row (row[:]) before passing it to is_match to prevent mutating self.raw_data. Creating a shallow copy of every row prior to condition evaluation causes severe memory and performance regressions. Evaluate the condition on the original row and only append a copy of it upon a successful match (filtered.append(original_row[:])), preserving the original list comprehension's evaluation order and memory efficiency."
