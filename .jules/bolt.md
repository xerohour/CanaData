## 2024-05-18 - Concurrency and CPU Processing Optimizations

**Learning:** During profiling and testing, the application suffered from thread contention and high CPU/Memory overhead. The root cause for the contention was a single shared `_menu_data_lock` in `CanaData.py`, which caused threads to wait sequentially even when updating disparate, non-overlapping collections. Furthermore, the `OptimizedDataProcessor.py` fallback flattening pipeline exhibited high Pandas overhead and memory allocation due to using `.copy()` in loops and applying Python logic (`json.dumps`) via `pd.Series.apply`.

**Action:** Replaced the single `_menu_data_lock` in `CanaData.py` with five fine-grained locks (`_items_lock`, `_empty_lock`, `_strains_lock`, `_count_lock`, `_locations_lock`), which isolates critical sections and improves multi-threaded horizontal scalability during network I/O. For `OptimizedDataProcessor.py`, eliminated `.copy()` overhead by converting standard explicit loops to fast list comprehensions and dictionary unpacking. Replaced Pandas Series mapping overhead (`df[col].apply`) by applying pure-Python list comprehensions directly over the target columns. These changes yielded significant improvements in local thread concurrency and optimized high-scale data manipulations.
