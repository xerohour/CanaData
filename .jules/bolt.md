## 2024-04-01 - List Comprehension O(num_filters * num_rows) String Join Bottleneck
**Learning:** In `CanaParse.py`, the `is_match` method was repeatedly performing expensive string joins (`" ".join([str(x) for x in row]).lower()`) for every single row, across *every single filter*. This creates an `O(num_filters * num_rows)` bottleneck. During benchmark tests on 5,000 items with 100 filters, this accounted for a large portion of the execution time, taking over 2 seconds.
**Action:** When filtering a dataset multiple times (e.g., applying multiple `FlowerFilter` configurations), pre-compute expensive per-row transformations (like creating a single joined string for word matching) outside the filter loop (`O(num_rows)`). Pass these pre-computed values to the condition function using `zip()`. This drops the execution time by >50%. Ensure the condition function continues to mutate the original row references correctly if downstream logic depends on it.
