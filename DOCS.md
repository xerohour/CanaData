# CanaData Usage Documentation

Welcome to the advanced documentation for CanaData. This guide provides code examples and detailed explanations for using the scraper and the parser.

## 🚀 Scaper: `CanaData.py`

The scraper is designed to efficiently collect data from Weedmaps. It now supports environment variables for configuration.

### Basic Usage

To run the scraper for a specific city:
```bash
python CanaData.py -go "los-angeles"
```

### GUI Usage

For a desktop workflow (Windows/macOS/Linux), run:

```bash
python scripts/gui_app.py
```

The GUI runs CanaData in a background thread, streams logs live, and keeps the app responsive while scraping.

### Configuration via `.env`

You can customize the scraping behavior by creating a `.env` file in the root directory:

```env
WEEDMAPS_BASE_URL=https://api-g.weedmaps.com/discovery/v1/listings
PAGE_SIZE=100
LOG_LEVEL=DEBUG
```

### Advanced Scraping Options

- **`-all`**: Scrapes all 50 states defined in `states.txt`.
- **`-mylist`**: Scrapes a custom list of slugs from `mylist.txt`.
- **`-tshoot`**: Enables troubleshooting mode (sets `LOG_LEVEL` to `DEBUG`).

## 🔍 Parser: `CanaParse.py`

The parser filters the CSV data generated by the scraper into beautiful HTML reports.

### Filtering Logic

Filters are defined in `parse-script/flower-filters.json`. Here is an example of a filter for high-quality concentrates:

```json
{
  "name": "Premium Concentrates",
  "key": "prices.gram",
  "compare": "<=",
  "price": 50,
  "categories": ["Concentrate"],
  "thc_floor": 80,
  "bad_words": ["vape", "cartridge"]
}
```

### Running the Parser

```bash
cd parse-script
python CanaParse.py
```

### Output

- **HTML Report**: Found in `parse-script/output/flower-filter.html`.
- **Email Report**: Optimized for mobile and email clients in `parse-script/output/flower-filter-email.html`.

## 🛠️ Development & Testing

### Running Tests

We use `pytest` for unit testing. To run the tests:

```bash
pytest tests/
```

### Logging

CanaData now uses Python's standard `logging` module. Logs will include timestamps and severity levels, making it easier to track progress and debug issues.

```
2026-02-17 12:00:00 - __main__ - INFO - Processing menu (1/10) --> listing-slug
2026-02-17 12:00:05 - __main__ - SUCCESS - Successfully exported (150 items) to CSV -> listing_results.csv
```
